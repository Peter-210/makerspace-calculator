\begin{DoxyItemize}
\item This project contains scripts that can run in different modes\+:
\begin{DoxyItemize}
\item {\ttfamily -\/p} (prod) -\/ Minimal installed tools on docker and optimized code from the compiler
\item {\ttfamily -\/d} (dev/debug) -\/ All tools installed for debugging the project (eg. GDB)
\item {\ttfamily -\/t} (test) -\/ Used for unit testing with GTest
\end{DoxyItemize}
\item To learn more about these modes, use the {\ttfamily -\/h} or {\ttfamily -\/-\/help} flag for the following scripts\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{./scripts/docker.sh\ -\/h}

\end{DoxyCode}

\item For any missing dependencies in docker container, run the following to clean container then re-\/run the script with the proper flag mode (-\/p/d)\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{./script/docker.sh\ -\/c}

\end{DoxyCode}

\item For any conflicts with docker container, run the following to clean container and image then re-\/run the script with the proper flag mode (-\/p/d)\+:
\begin{DoxyItemize}
\item The order is important. Clean container first, then image 
\begin{DoxyCode}{0}
\DoxyCodeLine{./script/docker.sh\ -\/c}
\DoxyCodeLine{./script/docker.sh\ -\/i}

\end{DoxyCode}

\end{DoxyItemize}
\item For CMake builds that are not updating to code changes, run the following to clean build then re-\/run the script with the proper flag mode (-\/p/d/t)\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{./script/make.sh\ -\/c}

\end{DoxyCode}

\item For GTest lib files being corrupted, run the following to clean lib files then re-\/run the script with the proper flag mode (-\/t)\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{./script/make.sh\ -\/l}
